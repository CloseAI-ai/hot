# HOT 8M 模型配置计划

## 目标

设计一个 8M 参数的 HOT 模型，确保：
1. 参数量精确控制在 8M 左右
2. 模型架构合理，符合行业最佳实践
3. 训练参数经过精心调整
4. 保持 HOT 的核心创新特性

## 一、参数量分析

### 当前 42M 模型参数分布

| 组件 | 参数量 | 占比 |
|------|--------|------|
| Embedding (= LM Head) | 20.10M | 47.1% |
| 12 x HOTLayer | 22.61M | 52.9% |
| **总计** | **42.72M** | 100% |

### 8M 模型设计约束

1. **Embedding 层占比大**：V=50257 时，Embedding = 50257 * D
2. **权重绑定**：LM Head 与 Embedding 共享，不重复计算
3. **HOT 特有参数极少**：每层仅 19 个标量参数

### 目标配置计算

**方案：D=128, L=4**

| 组件 | 公式 | 参数量 |
|------|------|--------|
| Embedding | 50257 * 128 | 6,432,896 |
| HOTLayer x 4 | 见下方计算 | ~1.5M |
| Final RMSNorm | 128 | 128 |
| theta_init | 1 | 1 |
| **总计** | | **~8.0M** |

**每层 HOTLayer 参数（D=128, H=8, d=16, F=512）：**

| 子模块 | 公式 | 参数量 |
|--------|------|--------|
| q_proj | 128 * 128 + 128 | 16,512 |
| k_proj | 128 * 128 + 128 | 16,512 |
| v_proj | 128 * 128 + 128 | 16,512 |
| o_proj | 128 * 128 + 128 | 16,512 |
| w1 (FFN) | 128 * 512 + 512 | 66,048 |
| w2 (FFN) | 512 * 128 + 128 | 65,664 |
| RMSNorm x 2 | 128 * 2 | 256 |
| HOT 组件 | 19 | 19 |
| **每层合计** | | **198,035** |

**4 层总计：198,035 * 4 = 792,140**

**模型总参数：6,432,896 + 792,140 + 128 + 1 = 7,225,165 ≈ 7.2M**

这个配置略低于 8M，让我调整。

## 二、最终配置设计

### 方案 A：D=128, L=5 (推荐)

| 组件 | 参数量 |
|------|--------|
| Embedding | 6,432,896 |
| HOTLayer x 5 | 990,175 |
| 其他 | 129 |
| **总计** | **7,423,200 ≈ 7.4M** |

### 方案 B：D=144, L=4

| 组件 | 参数量 |
|------|--------|
| Embedding | 7,236,048 |
| HOTLayer x 4 | ~900K |
| **总计** | **~8.1M** |

### 方案 C：D=160, L=3

| 组件 | 参数量 |
|------|--------|
| Embedding | 8,041,120 |
| HOTLayer x 3 | ~750K |
| **总计** | **~8.8M** |

**推荐方案 B：D=144, L=4**

理由：
1. 参数量最接近 8M
2. hidden_size=144 是 2 的幂次附近，计算效率高
3. 4层足够学习基础语言模式
4. head_dim=18 (144/8) 不是整数，需要调整

**调整为：D=144, H=9, d=16**

验证：9 * 16 = 144 ✓

## 三、最终模型配置

```yaml
model:
  hidden_size: 144
  num_heads: 9
  head_dim: 16
  num_layers: 4
  vocab_size: 50257
  ffn_size: 576        # 4 * hidden_size
  dropout: 0.1
```

**参数量验证：**

| 组件 | 参数量 |
|------|--------|
| Embedding | 50257 * 144 = 7,236,048 |
| Attention (每层) | 4 * (144*144 + 144) = 83,520 |
| FFN (每层) | 144*576 + 576 + 576*144 + 144 = 166,464 |
| RMSNorm (每层) | 144 * 2 = 288 |
| HOT (每层) | 19 |
| **每层合计** | **250,291** |
| **4 层总计** | **1,001,164** |
| Final RMSNorm | 144 |
| theta_init | 1 |
| **模型总计** | **8,237,357 ≈ 8.2M** |

## 四、训练配置优化

### 4.1 学习率策略

对于 8M 小模型，需要调整学习率：

```yaml
training:
  learning_rate: 5e-4          # 小模型可用更高学习率
  lr_schedule: "cosine"
  warmup_steps: 1000           # 减少预热步数
  min_lr_ratio: 0.1            # 最小学习率比例
```

**理由：**
- 小模型收敛更快，不需要太长预热
- 更高学习率有助于跳出局部最优
- cosine 调度保持稳定性

### 4.2 Batch Size 策略

```yaml
training:
  batch_size: 32               # 增大 batch size
  gradient_accumulation: 2     # 有效 batch = 64
  max_steps: 150000            # 增加训练步数
```

**理由：**
- 小模型显存占用低，可用更大 batch
- 更大 batch 提供更稳定的梯度估计
- 增加步数弥补参数量减少

### 4.3 正则化策略

```yaml
training:
  weight_decay: 0.1
  max_grad_norm: 1.0
  dropout: 0.1
```

**理由：**
- 小模型需要适当正则化防止过拟合
- 梯度裁剪保持训练稳定
- dropout 率保持与 42M 模型一致

### 4.4 退火策略

```yaml
hot:
  annealing:
    schedule: "cosine"
    warmup_steps: 1500         # 缩短退火预热
```

**理由：**
- 小模型学习 phase dynamics 更快
- 缩短预热让模型更早进入 full gating

## 九、风险评估

### 潜在风险

1. **模型容量不足**
   - 风险：无法学习复杂模式
   - 缓解：增加训练步数，监控验证损失

2. **过拟合**
   - 风险：小模型更容易过拟合
   - 缓解：保持 dropout，使用验证集早停

3. **训练不稳定**
   - 风险：小模型对学习率更敏感
   - 缓解：使用 warmup，监控梯度范数

4. **HOT 特性不明显**
   - 风险：参数太少可能无法充分利用相位动力学
   - 缓解：监控相位分布，确保非平凡
