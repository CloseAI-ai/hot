# HOT 模型配置对比

## 概述

本文档对比 HOT 42M 和 HOT 8M 两种模型配置，说明设计决策和权衡。

## 模型架构对比

### 配置参数

| 参数 | HOT 42M | HOT 8M | 说明 |
|------|---------|--------|------|
| hidden_size | 400 | 120 | 隐藏层维度 |
| num_heads | 8 | **6** | 注意力头数 (关键修改) |
| head_dim | 50 | **20** | 每头维度 (关键修改) |
| num_layers | 12 | 10 | Transformer 层数 |
| vocab_size | 50257 | 50257 | 词汇表大小 (GPT-2) |
| ffn_size | 1550 | 480 | FFN 中间层维度 |
| dropout | 0.1 | 0.1 | Dropout 比率 |
| normalization | Pre-LN | Pre-LN | 前置归一化 (稳定训练) |

### 参数量分布

| 组件 | HOT 42M | HOT 8M | 占比 (8M) |
|------|---------|--------|-----------|
| Embedding | 20.10M | 6.03M | 77.6% |
| Attention (每层) | 0.64M | 0.05M | - |
| FFN (每层) | 1.24M | 0.10M | - |
| Transformer 层总计 | 22.61M | 1.74M | 22.4% |
| **总计** | **42.72M** | **7.77M** | 100% |

### HOT 特有组件

| 组件 | 每层参数 | 作用 |
|------|----------|------|
| IntrinsicFrequency | 16 | Q/K 能量差 → 频率 |
| PhaseDynamics | 1 | 可学习时间步长 dt |
| PhaseGating | 1 | 耦合强度 alpha |
| gamma | 1 | 残差缩放系数 |
| **合计** | **19** | |

HOT 特有参数占总参数比例极小（<0.01%），但引入了可训练的相位隐状态。

## 训练配置对比

| 参数 | HOT 42M | HOT 8M | 调整原因 |
|------|---------|--------|----------|
| batch_size | 16 | 32 | 小模型显存占用低 |
| gradient_accumulation | 3 | 2 | 有效 batch 64 vs 48 |
| max_steps | 100,000 | 150,000 | 小模型需要更多步数 |
| learning_rate | 3e-4 | 4e-4 | 保守学习率，避免梯度饱和 |
| warmup_steps | 2,000 | 1,500 | 适中预热，平衡稳定性和效率 |
| annealing warmup | 2,000 | 1,500 | 缩短退火预热 |

### 学习率调整说明

**原计划**: learning_rate=5e-4, warmup_steps=1000

**风险分析**:
- LR 提升 66% (3e-4 → 5e-4) + Warmup 砍半 (2K → 1K)
- 小模型初始化更"脆弱"，前 1K 步直接施加 5e-4 极易导致:
  - Attention Softmax 梯度进入饱和区
  - 早期训练震荡 (Loss 尖峰)

**调整方案**:
- learning_rate: 5e-4 → 4e-4 (降低 20%)
- warmup_steps: 1000 → 1500 (增加 50%)
- 保持 max_grad_norm=1.0 (梯度裁剪)

**预期效果**:
- 更平滑的 Loss 曲线
- 避免早期训练不稳定
- 保持小模型的训练效率

### 权重衰减调整说明

**原计划**: weight_decay=0.1

**风险分析**:
- 10层小网络（7.77M）在仅约5.3个Epoch的训练中
- 0.1的强衰减会强制将大量权重推向零
- 严重抑制模型的容量表达，导致Loss长期居高不下

**调整方案**:
- weight_decay: 0.1 → 0.01 (AdamW黄金默认值)

**预期效果**:
- 保持模型容量表达
- 避免权重过度衰减
- 更好的训练收敛

### 最小学习率调整说明

**原计划**: min_lr_ratio=0.1 (最终LR=4e-5)

**优化建议**:
- min_lr_ratio: 0.1 → 0.05 (最终LR=2e-5)
- 让模型在最后30K步更精细地"微调"10层网络的深层表征
- 通常能带来0.5%~1%的验证集精度提升

**预期效果**:
- 更精细的深层表征微调
- 验证集精度提升
- 更好的泛化能力

## 关键修复说明

### 修复 1：注意力头数 (最高优先级)

**问题**：
- 原配置：num_heads=8, head_dim=15 (120/8=15)
- 风险：15维的点积结果极易落入饱和区，导致梯度消失

**修复**：
- 新配置：num_heads=6, head_dim=20 (120/6=20)
- 效果：20维是能让10层深网络稳定传递梯度的生死线

**原理**：
- 点积注意力的梯度与 head_dim 成正比
- head_dim=15 时，梯度信号在10层传播中会指数衰减
- head_dim=20 提供足够的梯度流，确保深层学习

### 修复 2：Pre-LN (前置归一化)

**问题**：
- 10层网络（隐藏层仅120维）的梯度流非常脆弱
- 标准的 Post-LN 在深层中极易引发梯度爆炸或消失

**状态**：
- ✅ 代码已使用 Pre-LN (hot_layer.py 第206、275行)
- norm1 在 Attention 之前
- norm2 在 FFN 之前

**效果**：
- Pre-LN 是10层小网络稳定训练的"保命符"
- 确保梯度在深层网络中稳定传播

## 结构优化说明

### 问题：Embedding 占比过高

初始设计 (hidden=144, layers=4):
- Embedding: 87.9%
- Transformer: 12.1%
- 问题: 4层网络表达力不足，本质上是"带极浅层语义融合的查找表"

### 解决方案：降低 Embedding 维度，增加 Transformer 深度

优化设计 (hidden=120, layers=10):
- Embedding: 77.6% (降低 10.3%)
- Transformer: 22.4% (提升 10.3%)
- 层数: 4 → 10 (增加 150%)

### 优化效果

1. **更强的表达能力**: 10层 Transformer 能学习更复杂的上下文关系
2. **更好的泛化能力**: 更多参数用于"逻辑推理"而非"查找"
3. **更深的语义理解**: 多层堆叠逐级抽象
4. **保持参数效率**: 总参数量仍为 ~8M

### 权衡

- **优势**: 更强的推理能力，更好的泛化
- **劣势**: 每层宽度略窄 (120 vs 144)，单层容量略低
- **结论**: 深度比宽度更重要，10层 120维优于 4层 144维

## 设计决策

### 1. 为什么选择 hidden_size=120？

- 120 = 8 heads * 15 dim，整除关系
- 降低 Embedding 参数占比 (87.9% → 77.6%)
- 为更多 Transformer 层腾出参数预算

### 2. 为什么选择 num_layers=10？

- 深度比宽度更重要（行业共识）
- 10层能学习更复杂的上下文关系
- 保持 HOT 的跨层相位传播特性
- 每层 174K 参数，总 Transformer 1.74M

### 3. 为什么增加 batch_size？

- 小模型显存占用低，可用更大 batch
- 更大 batch 提供更稳定的梯度估计
- 有效 batch 64 vs 48，训练更稳定

### 4. 为什么提高学习率？

- 小模型收敛更快，需要更高学习率跳出局部最优
- 5e-4 是小模型的常用学习率
- 配合 warmup 确保训练稳定

### 5. 为什么增加训练步数？

- 参数量减少 5倍，需要更多步数学习
- 150K 步确保充分收敛
- 监控验证损失决定是否早停

## 使用方法

## 监控指标

训练过程中需要特别关注：

1. **训练/验证损失**：确保正常收敛
2. **学习率曲线**：验证 cosine 调度
3. **退火 beta**：确保按计划预热
4. **梯度范数**：监控训练稳定性
5. **相位分布**：确保 HOT 特性生效
6. **吞吐量**：验证速度提升预期
